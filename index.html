import numpy as np
from scipy import stats
scores = np.array([45,52,48,61,55,999,43,58,47,-200,50])

# ① IQR 法（不假設分布，適合偏態資料）
Q1, Q3 = np.percentile(scores, [25, 75])
IQR       = Q3 - Q1
lower, upper = Q1 - 1.5*IQR, Q3 + 1.5*IQR
iqr_mask  = (scores < lower) | (scores > upper)   # True = 異常

# ② Z-Score 法（假設常態分布，|z|>3 為異常）
z_scores  = np.abs(stats.zscore(scores))
z_mask    = z_scores > 3

# ③ Winsorization（截尾處理，保留樣本數）
cleaned = np.clip(scores, lower, upper)
💡 iPAS 考試重點： IQR 法不假設分布、適合偏態資料；Z-Score 法假設常態分布，企業常用 |z|≥3 作為閾值。np.clip() 即 Winsorization 的 Python 實作。

```
📝 關卡二考題 · 資料縮放
Q4 在建立 KNN（K最近鄰）分類模型前，為什麼需要對特徵進行資料縮放？
✅ 正確！ KNN 使用歐氏距離計算相似度。若年齡（0-100）與薪資（0-500,000）同時作為特徵，薪資的數值尺度遠大於年齡，導致距離計算幾乎完全由薪資主導，年齡幾乎無影響。縮放可確保每個特徵平等貢獻。
❌ 錯誤。 資料縮放不直接解決過擬合問題（那是正則化、Dropout的工作）。縮放的目的是讓不同尺度的特徵在距離計算中有公平的貢獻。
❌ 錯誤。 縮放對資料讀取速度無明顯影響。正確答案是B——避免大尺度特徵主導距離計算。
❌ 錯誤。 資料縮放不增加樣本數量，那是資料增強（Data Augmentation）的功能。
Q5 某資料集的薪資欄位含有極端值（如 CEO 薪資遠高於一般員工），以下哪種縮放方法最適合？
✅ 正確！ Robust Scaler 使用中位數（Q2）和四分位距（IQR = Q3 - Q1）進行縮放，不受極端值影響。計算公式：x' = (x - Q2) / IQR。這讓絕大多數「正常」員工的薪資能夠合理縮放，CEO 的極端值不會扭曲整體分布。
❌ 錯誤。 Min-Max 對極端值高度敏感——CEO 的超高薪資會成為 max，讓其他員工的縮放值都壓縮到接近 0，失去區分意義。
❌ 錯誤。 Z-score 標準化 使用均值和標準差，極端值會拉高均值並增大標準差，雖比 Min-Max 好但仍受影響，不如 Robust Scaler。
❌ 錯誤。 若後續使用 KNN、SVM、神經網路等演算法，縮放是必要的。
Q6 以下關於資料標準化（Z-score 標準化）的敘述，哪項正確？
✅ 正確！ Z-score 標準化是線性變換，只改變資料的均值（→0）和標準差（→1），不改變分布形狀。原本右偏的資料標準化後仍然右偏。且值域可能超出 [0,1]（例如極端值的 z-score 可能達 3 以上）。
❌ 錯誤。 Z-score 標準化 不改變分布形狀。若原始資料是左偏或雙峰分布，標準化後仍保持相同形狀。
❌ 錯誤。 [0,1] 是 Min-Max 正規化的特性。Z-score 標準化 的值域是(-∞, +∞)，大多數值在 [-3, 3] 之間。
❌ 錯誤。 決策樹基於特徵分裂點，對資料尺度不敏感，不需要縮放。

# ── 關卡二：Scikit-learn 資料縮放三法 ───────────────────────
```

from sklearn.preprocessing import MinMaxScaler, StandardScaler, RobustScaler
from sklearn.model_selection import train_test_split
import numpy as np

X = np.array([[38, 1500, 4000],   # [年齡, 力量指數, 訓練時數]
[17, 7500, 800 ],
[45, 9800, 200 ]])

X_train, X_test = train_test_split(X, test_size=0.2, random_state=42)

# ① Min-Max 正規化 → 值域 [0, 1]
mm = MinMaxScaler()
X_train_mm = mm.fit_transform(X_train)   # fit + transform 訓練集
X_test_mm  = mm.transform(X_test)        # 只 transform 測試集！

# ② Z-score 標準化 → μ=0, σ=1
std = StandardScaler()
X_train_std = std.fit_transform(X_train)

# ③ Robust Scaler → 抗離群值（使用 Q2、IQR）
rob = RobustScaler()
X_train_rob = rob.fit_transform(X_train)

# ⚠️ 黃金法則：scaler 只 fit 訓練集，再 transform 測試集
#    若對測試集也 fit_transform → 資料洩漏（Data Leakage）！
💡 iPAS 考試重點： 縮放器必須只對訓練集呼叫 fit_transform()，測試集只能呼叫 transform()，否則測試集的統計資訊會洩漏到訓練過程，造成資料洩漏（Data Leakage），高估模型表現。

```
Q7 PCA 降維前為何必須先對資料進行標準化？
✅ 正確！ PCA 基於方差最大化。若特徵尺度不一致（如薪資的方差遠大於年齡），PCA 會將最大方差的方向（薪資方向）作為 PC1，完全忽略其他特徵的貢獻。標準化確保每個特徵平等參與主成分的計算。
❌ 錯誤。 標準化（Z-Score）不改變分布形狀，不能讓偏態變常態。且PCA不要求常態分布。
❌ 錯誤。 PCA 可以處理連續型浮點數資料，沒有整數要求。
❌ 錯誤。 標準化不降低計算複雜度，目的是消除尺度差異對主成分計算的影響。
Q8 進行 PCA 後，各主成分的解釋方差比（Explained Variance Ratio）依次為： PC1=45%, PC2=25%, PC3=18%, PC4=12%。若要保留至少 85% 的資訊，應選幾個主成分？
✅ 正確！
PC1 = 45%（累積 45%，未達 85%）
PC1+PC2 = 70%（累積 70%，未達 85%）
PC1+PC2+PC3 = 88%（累積 88%，超過 85%！）
因此選擇 3 個主成分，從 4 維降至 3 維，保留 88% 的資訊。
❌ 錯誤。 45% < 85%，資訊損失過多。
❌ 錯誤。 70% < 85%，仍未達標準。
❌ 錯誤。 保留全部特徵就沒有降維，失去 PCA 的意義。
Q9 以下關於 PCA 與 LDA 的比較，哪項敘述正確？
✅ 正確！
• PCA（主成分分析）：無監督降維，最大化方差，不使用類別標籤
• LDA（線性判別分析）：監督式降維，最大化類間距離/最小化類內距離，需要類別標籤
• LDA 更適合作為分類任務的前處理步驟
• LDA 最多降至 (類別數-1) 個維度
❌ 錯誤。 PCA 是無監督的（不需要標籤），LDA 才是監督式的。
❌ 錯誤。 LDA 最多降至（類別數-1）個維度，不能超過原始特徵數，且通常遠少於原始特徵數。
❌ 錯誤。 PCA 只能處理線性關係。非線性降維應使用 Kernel PCA、t-SNE 或 UMAP。
```

from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
import numpy as np
import matplotlib.pyplot as plt

X = np.random.randn(1000, 100)   # 1000 樣本，100 特徵

# ① 必須先標準化！PCA 對特徵尺度高度敏感
scaler = StandardScaler()
X_std  = scaler.fit_transform(X)

# ② 初步 PCA：保留全部成分，查看解釋方差
pca_full = PCA()
pca_full.fit(X_std)
evr     = pca_full.explained_variance_ratio_   # 各 PC 解釋比例
cum_evr = np.cumsum(evr)                       # 累積解釋比例

# ③ 找出保留 95% 方差所需的成分數
n_comp = np.argmax(cum_evr >= 0.95) + 1
print(f”保留 95% 方差需要 {n_comp} 個主成分”)

# ④ 執行降維
pca   = PCA(n_components=n_comp)
X_pca = pca.fit_transform(X_std)
print(f”維度：{X.shape[1]} 維 → {X_pca.shape[1]} 維”)

# ⑤ Scree Plot（手肘圖）視覺化選擇成分數
plt.plot(np.cumsum(evr[:20]), marker=‘o’)
plt.axhline(y=0.95, color=‘r’, linestyle=’–’, label=‘95% threshold’)
plt.xlabel(‘Number of Components’)
plt.ylabel(‘Cumulative EVR’)
plt.legend(); plt.show()
💡 iPAS 考試重點： PCA 前必須標準化，否則大尺度特徵主導主成分方向。用 explained_variance_ratio_ 查看各 PC 貢獻，用 np.argmax(cumsum >= 0.95) 找到保留 95% 方差的成分數。也可直接設 PCA(n_components=0.95) 讓 sklearn 自動選擇。
